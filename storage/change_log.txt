# Change Log

## Backend (JWT customer auth)
- Added custom JWT guard using `SimpleJwtService` and request-based guard registration in `AppServiceProvider`.
- Introduced `CustomerJwtAuthController` with login/refresh/me/logout endpoints and wired customer routes to allow JWT without CSRF.
- Added middleware alias `customer.auth` to prefer JWT but allow session fallback; updated routes to use it for customer endpoints.
- Allowed Lighthouse to use `customer_jwt` guard and updated `myEvents` to accept JWT.
- Customer event mutation now authenticates via JWT guard (fallback to session).
- Added `customer` relation to `Events` model for dashboard listings.
- Added migration for rejected status; dashboard stats for customers/events and pending/upcoming sections.

## Frontend (React) switched to JWT
- `useCustomer` now stores JWT in localStorage, provides login/logout, and fetches user via `/customer/jwt/me`.
- `CustomerLogin` uses JWT login; `CustomerRegister` simplified (no Sanctum).
- `Nav` uses `useCustomer`; logout clears token.
- `EventDetail` uses JWT for ticket status/book/cancel; added countdown UI.
- `CreatedByMe`, `UpcomingEvents`, `CreateNew` send Authorization header for GraphQL; create-event upload no longer uses CSRF.
- `CustomerAccount` uses `useCustomer`; removed AuthContext usage and provider wrapper.
- Removed `AuthContext` entirely and its imports/wrapper.

## Dashboard tweaks
- Added 2x2 header grid with customer stats, event stats, pending list, and upcoming list; stats come from `DashboardController`.
- Events index: description column with modal; pending rows have approve/reject actions; sort links respect pending/approved routes.

## Routes/guards
- `/customer/jwt/login|refresh|logout|me` added; `/customer` group excludes CSRF.
- Customer booking/status endpoints now use `customer.auth` (JWT or session).
